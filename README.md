# PdfStamper — OutSystems ODC External Library

Stamp form data onto flat (non-AcroForm) PDF templates at precise coordinates.
Works with any PDF — government forms, contracts, NDAs, leases, and more.

## Actions

| Action | Description |
|--------|-------------|
| `StampPDF(templatePdf, fieldMapJson, valuesJson, inkColor)` | Stamp values onto a PDF template. Returns filled PDF as binary. |
| `GetPDFInfo(pdfData)` | Get page count + dimensions. Used by the Field Mapper. |
| `GetBuildVersion()` | Returns deploy metadata. Useful for verifying which version is running. |

## Field Map JSON

Generated by the Field Mapper tool. Defines where each field sits on each page.

```json
[
  { "field": "firstName",  "page": 0, "x": 95,  "y": 567, "type": "text",     "maxWidth": 150, "fontSize": 9 },
  { "field": "isLLC",      "page": 0, "x": 419, "y": 541, "type": "checkbox"                                 },
  { "field": "signDate",   "page": 5, "x": 450, "y": 221, "type": "text",     "maxWidth": 100, "fontSize": 9 }
]
```

- `page` — 0-based index
- `x`, `y` — PDF coordinate space (origin = bottom-left, y increases upward)
- `type` — `"text"` or `"checkbox"`
- `maxWidth` — max text width in points before truncation
- `fontSize` — font size in points (default: 9)

## Values JSON

Key-value pairs matching field names in the field map.

```json
{
  "firstName": "Jane Smith",
  "isLLC":     "true",
  "signDate":  "03/03/2026"
}
```

## Ink Color

R,G,B values 0–1. Examples:
- `"0,0,0"` — Black
- `"0,0,0.55"` — Dark blue (recommended for forms)
- `"0.8,0,0"` — Red

## Setup

### GitHub Secrets required

| Secret | Value |
|--------|-------|
| `ODC_PORTAL_URL` | Your ODC portal URL (no trailing slash) |
| `ODC_CLIENT_ID` | API client ID from ODC Portal → Settings → API Management |
| `ODC_CLIENT_SECRET` | API client secret |

### ODC API Client Permissions
- Asset management > Create
- Asset management > Read
- Asset management > Update

## Development Workflow

```
1. Edit code locally
2. git add + git commit + git push → triggers GitHub Actions
3. Actions builds, injects build metadata, deploys to ODC
4. If ReadyForReview: approve in ODC Portal → Libraries → External Libraries
5. Test in ODC
```
